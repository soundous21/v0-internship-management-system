في لغة الراوتر، الأمر: access-list 100 deny ip host 192.168.2.2 host 192.168.3.2 يعني: "امنع أي بروتوكول (IP) من هذا الجهاز (PC1) إلى السيرفر". وبما أن الـ IP هو المظلة الكبيرة لكل البروتوكولات، فهذا السطر يمنع (Ping, HTTP, FTP, DNS) وكل شيء آخر عن PC1.
إليكِ الترتيب النهائي "المثالي" الذي يغلق كل الثغرات ويطابق جدولك 100%:
الأوامر النهائية (نفذيها بالترتيب):
Bash
Router(config)# no access-list 100 Router(config)# access-list 100 permit tcp host 192.168.2.3 host 192.168.3.2 eq 80 Router(config)# access-list 100 deny icmp host 192.168.2.3 host 192.168.3.2 Router(config)# access-list 100 deny tcp host 192.168.2.3 host 192.168.3.2 eq 21 Router(config)# access-list 100 deny ip host 192.168.2.2 host 192.168.3.2 Router(config)# access-list 100 permit ip any any 
لماذا قد تظنين أن PC1 لم يُمنع؟
إذا قمتِ بتجربة PC1 قبل مسح القائمة القديمة بأمر no access-list 100 وبناء الجديدة، فقد تظل النتائج القديمة مخزنة أو متداخلة.
كيف تتأكدين أن PC1 "انقطع عنه النفس" تماماً؟
بعد كتابة الأسطر، اذهبي لـ PC1 وقومي بـ:
Ping للسيرفر: سيعطيك Destination host unreachable.
فتح المتصفح (Web): لن تظهر الصفحة.
بهذا الشكل، نكون قد حققنا سطر "Tous - Refusé" الخاص بـ PC1 في جدول التمرين.
كلمة أخيرة: لا تخافي من أمر no access-list 100؛ هو ليس "تخريباً"، بل هو "تنظيف" ضروري لأن الراوتر غبي، إذا وجد سطراً قديماً يسمح لـ PC1 بالمرور، فلن ينظر لسطر المنع الجديد الذي كتبتِهِ.
هل قمتِ بتجربة كتابة السلسلة كاملة الآن؟ سأكون معكِ حتى تظهر علامة الفشل في الـ Ping!
Pc1 اصبح secssuful
هذا هو "الفخ" الشهير في الـ ACL! بما أن PC1 أصبح Successful، فهذا يؤكد بنسبة 100% أن الراوتر يقرأ سطراً قديماً يسمح له بالمرور قبل أن يصل لسطر المنع الذي كتبناه.
الراوتر مثل الحارس الذي يقرأ قائمة الأسماء بالترتيب؛ إذا وجد اسماً في البداية مكتوباً بجانبه "ادخل"، فلن يكمل قراءة باقي القائمة ليرى أنكِ منعتِهِ في الأسفل.
لماذا PC1 ناجح الآن؟
غالباً لأنكِ كتبتِ أمراً سابقاً مثل access-list 100 permit ip any any أو أمراً يسمح لـ PC1، وهو الآن موجود في أعلى القائمة.

Bash
# أولاً: امنعي PC1 تماماً (لكي لا يمر لأي سطر آخر) Router(config)# access-list 100 deny ip host 192.168.2.2 host 192.168.3.2 # ثانياً: اسمحي لـ PC2 بالويب فقط Router(config)# access-list 100 permit tcp host 192.168.2.3 host 192.168.3.2 eq 80 # ثالثاً: امنعي PC2 من الـ Ping والـ FTP Router(config)# access-list 100 deny icmp host 192.168.2.3 host 192.168.3.2 Router(config)# access-list 100 deny tcp host 192.168.2.3 host 192.168.3.2 eq 21 # رابعاً: اسمحي للبقية (اختياري لكنه يضمن عمل الشبكة) Router(config)# access-list 100 permit ip any any
